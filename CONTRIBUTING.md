# Contributing

Thank you for considering contributing to this project! Every contribution matters, whether it is fixing a typo, reporting a bug, or proposing a new feature.

## How to Contribute

### Reporting Bugs

1. Check existing [Issues](../../issues) to avoid duplicates.
2. Open a new issue with a clear title and description.
3. Include steps to reproduce, expected behaviour, and actual behaviour.
4. Add relevant labels (e.g., `bug`).

### Suggesting Features

1. Open an issue with the `enhancement` label.
2. Describe the problem the feature would solve.
3. Outline your proposed solution.

### Submitting Changes

1. **Fork** the repository and create a new branch from the default branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes, keeping commits small and focused.
3. Follow the existing code style and conventions.
4. Write or update tests if applicable.
5. Ensure all tests pass before submitting.
6. **Push** your branch and open a **Pull Request**:
   - Provide a clear PR title and description.
   - Reference any related issues (e.g., `Closes #42`).

### Code Review

- All submissions require a pull request review before merging.
- Maintainers may request changes — this is a normal part of the process.
- Be responsive to feedback and open to suggestions.

## Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/<your-username>/<repo-name>.git
   ```
2. Follow the setup instructions in the project's `README.md`.

## Style Guidelines

- Write clear, descriptive commit messages.
- Keep pull requests focused on a single change.
- Document new features or changes in behaviour.

## Code of Conduct

By participating in this project you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
